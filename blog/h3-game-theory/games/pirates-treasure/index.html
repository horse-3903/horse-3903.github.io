Setup

Definition:

Sequential Bargaining and Voting Game

  • Players: Five pirates ordered by seniority: AAA, BBB, CCC, DDD, and EEE.
  • Strategies: A proposer chooses an allocation of 100 coins among the survivors; Each voter chooses Yes or No after seeing the proposal; Preferences are lexicographic: Survival, Coins, Fewer Rivals.

Rules

  • Start with five pirates dividing 100100100 coins by seniority; Players alternate proposing allocations and voting Yes or No among survivors.
  • The player who forms a passing coalition survives with the proposed allocation.
  • If a proposal passes, the game ends; If it fails, the proposal is killed and the next pirate proposes.
  • If a tie in payoffs occurs, the proposer is accepted
  • With nnn survivors, the proposer needs only enough votes to avoid losing.

Game Tree

Legend:

  • Pass (P): Proposal accepted
  • Fail (F): Proposer killed

Derivation (Backward Induction)

Step 1: Only EEE survives

  • No vote is needed; EEE keeps all 100 coins, so the rollback outcome is (0,0,0,0,100)(0,0,0,0,100)(0,0,0,0,100).

Step 2: DDD proposes to EEE

  • If DDD is killed, Step 1 gives E=100E=100E=100.
  • DDD votes Yes on his own proposal; a 111-111 tie passes, so DDD needs no extra vote.
  • DDD maximises own coins by proposing (0,0,0,100,0)(0,0,0,100,0)(0,0,0,100,0); EEE can vote No, but the proposal still passes.

Step 3: CCC proposes to D,ED,ED,E

  • If CCC is killed, Step 2 gives continuation payoffs (0,0,0,100,0)(0,0,0,100,0)(0,0,0,100,0).
  • DDD gets 100 after rejection, so rejects any offer below 100; EEE gets 0, so 1 coin makes EEE prefer acceptance.
  • With three survivors, CCC needs one extra vote besides his own; the cheapest pivotal vote is EEE for 1 coin.
  • CCC proposes (0,0,99,0,1)(0,0,99,0,1)(0,0,99,0,1).

Step 4: BBB proposes to C,D,EC,D,EC,D,E

  • If BBB is killed, Step 3 gives continuation payoffs (0,0,99,0,1)(0,0,99,0,1)(0,0,99,0,1).
  • CCC gets 99 after rejection, so needs more than 99; EEE gets 1, so needs more than 1.
  • DDD gets 0 after rejection, but 0 now ties on coins and rejection leaves fewer rivals, so DDD needs 1.
  • With four survivors, BBB needs one extra vote because a 222-222 tie passes; the cheapest pivotal vote is DDD for 1 coin.
  • BBB proposes (0,99,0,1,0)(0,99,0,1,0)(0,99,0,1,0).

Step 5: AAA proposes to B,C,D,EB,C,D,EB,C,D,E

  • If AAA is killed, Step 4 gives continuation payoffs (0,99,0,1,0)(0,99,0,1,0)(0,99,0,1,0).
  • BBB gets 99 after rejection, so needs more than 99; DDD gets 1, so needs more than 1.
  • CCC and EEE each get 0 after rejection, so each can be bought with 1 coin.
  • With five survivors, AAA needs two extra votes besides his own; the cheapest pivotal votes are CCC and EEE for 1 coin each.
  • AAA proposes (98,0,1,0,1)(98,0,1,0,1)(98,0,1,0,1).

Derivation (Nash Equilibrium)

  • With all five pirates, AAA needs two extra votes.
  • The cheapest votes are CCC and EEE, so AAA proposes
(98,0,1,0,1)(98,0,1,0,1)(98,0,1,0,1)
  • AAA, CCC, and EEE vote Yes, so the proposal passes 333-222.

Nash Equilibrium

Result:

The rollback equilibrium proposal is

{A,B,C,D,E}↦{98,0,1,0,1}\{A,B,C,D,E\} \mapsto \{98,0,1,0,1\}{A,B,C,D,E}↦{98,0,1,0,1}

It passes immediately with votes from AAA, CCC, and EEE.

Social Optimum

  • Total coins always sum to 100.
  • The game is about survival and distribution, not surplus creation.

Insights

Insight:

  • Backward induction turns the five-pirate problem into a chain of smaller voting games.
  • A proposer buys the cheapest pivotal votes, not the friendliest pirates.
  • The tie rule is decisive because it lowers the number of votes the proposer must buy.
