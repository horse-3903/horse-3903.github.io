RNNs, LSTMs, and GRUs

Syllabus Map

  • Study map: Syllabus Study Map

Outline

  • Sequence modelling is a machine learning technique for analysing and predicting patterns in ordered data
  • This means the sequence (like words in a sentence or stock prices over time) matters
  • These sequential models typically consist of:
    • RNNs
    • LSTMs
    • GRUs

RNNs

Core idea (high level)

  • RNNs process sequences step by step, carrying a hidden state forward
  • Each timestep reuses the same weights to incorporate past context
  • This simple memory can struggle with long-term dependencies

Notation

  • Input at time ttt: xtx_txt​
  • Hidden state (memory) at time ttt: ata_tat​
  • Output at time ttt: yty_tyt​
  • Previous hidden state: at−1a_{t-1}at−1​

How it works (specific)

  • RNNs utilise recurrent connections, where the output of a neuron at one time step is fed back as input to the network at the next time step
  • For a RNN, there are a few things we need to keep track of:
    • The current timestep ttt
    • The input at the current timestep xtx_txt​
    • The output at the current timestep yty_tyt​
    • The hidden states at the previous timestep at−1a_{t-1}at−1​ and current timestep ata_tat​
  • Broadly speaking, the RNN can be summarised as a function
fθ:(xt,at−1)↦(yt,at)f_\theta: (x_t, a_{t-1}) \mapsto (y_t, a_t)fθ​:(xt​,at−1​)↦(yt​,at​)

1. Set an initial hidden state a0a_0a0​

  • The initial hidden state a0a_0a0​ needs to be passed into the first timestep t=1t=1t=1
  • Usually, this is done by setting a0=0a_0 = 0a0​=0
  • Some implementations learn a trainable a0a_0a0​ instead of using zeros
  • The choice of a0a_0a0​ can affect early outputs for short sequences

2. Calculate the hidden state ata_tat​

  • Using the hidden state of the previous timestep t−1t-1t−1, take an input xtx_txt​ to calculate ata_tat​
  • The nonlinearity ϕ\phiϕ is commonly tanh⁡\tanhtanh or ReLU\mathrm{ReLU}ReLU
  • Weights are shared across all timesteps, which keeps parameter count fixed
at=ϕ(Waat−1+Wxxt+ba)a_t = \phi(W_a a_{t-1} + W_x x_t + b_a)at​=ϕ(Wa​at−1​+Wx​xt​+ba​)

3. Calculate the output for this timestep yty_tyt​

  • Using the hidden state ata_tat​, calculate the output yty_tyt​
  • The output can be used at every timestep or only at the final step, depending on the task
  • A softmax layer is common for classification outputs
yt=Wyat+byy_t = W_y a_t + b_yyt​=Wy​at​+by​

4. Pass the current hidden state ata_tat​ to the next timestep t+1t+1t+1

  • The current hidden state ata_tat​ is passed along to the next timestep as a form of input
  • Steps 1 to 4 are repeated for all of the sequential data
  • Backpropagation through time (BPTT) is used to train the parameters
  • Gradients can explode or vanish as sequence length grows

Practical usage

  • Short-sequence tasks where long-term memory is not critical (e.g., character-level text, simple sequence classification)
  • Baseline time-series forecasting or anomaly detection with limited context
  • Lightweight sequence labelling when model size and speed matter

LSTMs

Core idea (high level)

  • A standard RNN has one stream of memory that gets overwritten as new inputs arrive
  • An LSTM adds a separate, protected memory channel and explicit control mechanisms that decide:
    • what to keep,
    • what to discard,
    • what to output at each time step.
  • This gives superiour control over the memory provided to the model at every timestep, splitting the memory into long-term memory and short-term memory

Notation (gates and memory)

  • The gates are element-wise values in [0,1][0, 1][0,1] that control how much information is retained, added, or exposed:
    • Forget gate: ftf_tft​ (how much of Ct−1C_{t-1}Ct−1​ to keep)
    • Input gate: iti_tit​ (how much new information to write)
    • Output gate: oto_tot​ (how much of the cell state to expose as output)
  • Cell state (long-term memory): CtC_tCt​
  • Hidden state (short-term memory/output): hth_tht​
  • Input at time ttt: xtx_txt​

How it works (specific)

1. Set the initial states h0h_0h0​ and C0C_0C0​

  • Initialise the hidden state and cell state (often zeros).
  • Some implementations learn h0h_0h0​ and C0C_0C0​ as trainable parameters
  • These initial states seed the memory for the first timestep

2. Compute the forget gate ftf_tft​

  • The forget gate decides how much of Ct−1C_{t-1}Ct−1​ is retained
  • Values close to 0 drop old information; values close to 1 keep it
ft=σ(Wf[ht−1,xt]+bf)f_t = \sigma(W_f [h_{t-1}, x_t] + b_f)ft​=σ(Wf​[ht−1​,xt​]+bf​)

3. Compute the input gate iti_tit​ and candidate memory C~t\tilde{C}_tC~t​

  • The input gate controls how much new information is written to memory
  • The candidate memory proposes new content to add
it=σ(Wi[ht−1,xt]+bi)i_t = \sigma(W_i [h_{t-1}, x_t] + b_i)it​=σ(Wi​[ht−1​,xt​]+bi​) C~t=tanh⁡(WC[ht−1,xt]+bC)\tilde{C}_t = \tanh(W_C [h_{t-1}, x_t] + b_C)C~t​=tanh(WC​[ht−1​,xt​]+bC​)

4. Update the cell state CtC_tCt​

  • The cell state blends retained memory with new candidate content
  • This pathway helps gradients flow across long sequences
Ct=ft⊙Ct−1+it⊙C~tC_t = f_t \odot C_{t-1} + i_t \odot \tilde{C}_tCt​=ft​⊙Ct−1​+it​⊙C~t​

5. Compute the output gate oto_tot​

  • The output gate decides how much of the cell state is exposed
  • It controls the information sent to the next layer or timestep
ot=σ(Wo[ht−1,xt]+bo)o_t = \sigma(W_o [h_{t-1}, x_t] + b_o)ot​=σ(Wo​[ht−1​,xt​]+bo​)

6. Compute the hidden state hth_tht​

  • The hidden state is the short-term, exposed memory
  • It is what downstream layers typically consume
ht=ot⊙tanh⁡(Ct)h_t = o_t \odot \tanh(C_t)ht​=ot​⊙tanh(Ct​)

Practical usage

  • Long-range dependency tasks (e.g., document classification, language modelling, machine translation)
  • Speech recognition and synthesis, where temporal context matters over many steps
  • Time-series forecasting with seasonal or delayed effects

GRUs

Core idea (high level)

  • A GRU merges the LSTM’s cell state and hidden state into a single state hth_tht​
  • It uses two gates to control information flow:
    • Update gate ztz_tzt​ decides how much past information to keep
    • Reset gate rtr_trt​ decides how much past information to forget when proposing new content
  • This makes GRUs simpler and often faster than LSTMs while retaining strong performance

Notation (gates and memory)

  • Update gate: ztz_tzt​ (how much of ht−1h_{t-1}ht−1​ to keep)
  • Reset gate: rtr_trt​ (how much of ht−1h_{t-1}ht−1​ to use in candidate)
  • Hidden state (memory/output): hth_tht​
  • Candidate hidden state: h~t\tilde{h}_th~t​
  • Input at time ttt: xtx_txt​

How it works (specific)

1. Set the initial hidden state h0h_0h0​

  • Initialise the hidden state (often zeros).
  • Some implementations learn a trainable h0h_0h0​
  • This state acts as the initial memory before any inputs

2. Compute the update gate ztz_tzt​

  • The update gate controls how much of the past is carried forward
  • Larger ztz_tzt​ values preserve ht−1h_{t-1}ht−1​
zt=σ(Wz[ht−1,xt]+bz)z_t = \sigma(W_z [h_{t-1}, x_t] + b_z)zt​=σ(Wz​[ht−1​,xt​]+bz​)

3. Compute the reset gate rtr_trt​

  • The reset gate controls how much past information influences the candidate
  • Smaller rtr_trt​ values push the model to ignore old context
rt=σ(Wr[ht−1,xt]+br)r_t = \sigma(W_r [h_{t-1}, x_t] + b_r)rt​=σ(Wr​[ht−1​,xt​]+br​)

4. Compute the candidate hidden state h~t\tilde{h}_th~t​

  • The candidate mixes current input with a gated version of past state
  • It represents the proposed new content for memory
h~t=tanh⁡(Wh[rt⊙ht−1,xt]+bh)\tilde{h}_t = \tanh(W_h [r_t \odot h_{t-1}, x_t] + b_h)h~t​=tanh(Wh​[rt​⊙ht−1​,xt​]+bh​)

5. Update the hidden state hth_tht​

  • The final state interpolates between old and candidate states
  • This interpolation is simpler than the LSTM cell/hidden split
ht=(1−zt)⊙h~t+zt⊙ht−1h_t = (1 - z_t) \odot \tilde{h}_t + z_t \odot h_{t-1}ht​=(1−zt​)⊙h~t​+zt​⊙ht−1​

Practical usage

  • Similar to LSTMs but with fewer parameters, useful for smaller datasets or faster training
  • Real-time or edge deployments where model size/latency is constrained
  • Robust baselines for many sequence tasks (e.g., sentiment, tagging, forecasting)
